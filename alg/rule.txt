подсчет количества
	подготовка к счету
		определяем группу подсчета
			определяем условия (границы) подсчета
				создаем группу объектов направления и границ (связь с которыми других объектов относит их к границе поиска группы счета)
				создаем объект общности (сравнив с которым очередной объект, относим его к группе счета)
				создаем объект отличия (сравнив с которым очередной объект группы счета, даем ему имя счета, учитываем его в счете)
				создаем объект завершения (сравнив с которым очередной объект группы счета или объект время, останавливаем счет)
		получаем первый объект в цепочке направления и границ
		
	счет (пока не завершен счет)
		получаем объект, связанный с объектом направления и границ
		сравниваем объект с объектом общности
		если объект совпадает, относим объект к группе счета
		сравниваем объект с предыдущи объектом в группе счета
		если объект отличается, даем ему уникальное имя (номер) в группе счета
		сравниваем объект с объектом завершения
		если объект совпадает, завершаем счет
		сравниваем объект текущего времени с объектом завершения
		если объект совпадает, завершаем счет
		получаем следующий объект в цепочке направления и границ
		если цепочка закончена, завершаем счет

	завершаем счет
		уникальное имя (номер) последнего объекта в группе счета несет в себе смысл количества объектов в группе счета в условиях объекта завершения
		создаем объект количества, связываем его с объектом группой счета, чтобы получать количество объектов в группе в будущем

мозг
	сначала
		вопринимает поток информации и ничего не выделяет (не понимает)
	по мере поступления информации
		выделяет совпадающие (по измерениям) объекты (части большого объекта), запоминает общее, группирует (создает новый логический объект), дает ему значение (имя)
		выделяет несовпадающие с общими уже выделенными в группы объектами, выделяя отличия
	
	зрение
		сначала темные и светлые пятна (2 объекта)
		потом по степени затемнения создаются еще объекты и так далее
		темное пятно - меньше информации, меньше нагрузка, меньше напряжения
		светлое пятно - много информации и напряжения
		сначала старается избегать светлого (закрывает глаза, смотрит в более темные участки)
		по мере привыкания, разбирает на объекты более светлые объекты, так как они более информативны, больше привлекают внимание
		
	в мозге встроен механизм потребности в информации, в анализе, в создании новых связей, в обучении, в выживании
	отсюда и стремление получать информацию, при полном противоречии, так как информация - это напряжение
	этот механизм заставляет нас через боль (большой поток информации напрямую мозгу, короткое замыкание) изучать мир, двигаться, выживать
	возможно этот механизм приобретенный человеком в процессе эволюции, необходимости выживать и совершенствовать свою деятельность
	все завязано на страхе боли
		боль вызывают
			сильные звуки
			тактильные температурные перегрузки
			тактильные механические перегрузки
			яркий свет
			сильное воздействие на вкусовые рецепторы
			едкий запах
			уменьшение поступления кислорода
		боль вызывает перегрузку мозга, которая вызывает страх потери контроля и страх смерти
		отсюда потребность контролировать мозг
		осознание боли - осознание снижения производительности мозга, снижение защиты, возможная смерть и потеря всех объектов (ценностей)
	страх
		создается и записывается при перегрузках мозга, осознании боли и потери контроляё
		объект, с которым связаны объекты условия, при которых были вызваны перегрузки и боль
		сравнение с этим объектом в обычное время вызывает реакцию организма и соответственно слабые перегрузки, осознание боли, вызывая еще больший страх и ступор
		
сравнение объектов
	любые связанные объекты являются друг для друга признаками (классами), данная связь определяет их схожесть между собой по этому признаку (принадлежность к классу)
	сам процесс сравнения - есть получение информации (перенос внимания) об объектах сравнения и поиск связей их с общим признаком сравнения (классом)

	1 вариант : сравнение конкретных объектов по признаку одного из объектов (поиск признака первого объекта у второго)
		выбрать первый объект для сравнения
		выбрать для сравнения объект-признак (класс) связанный с первым объектом
		выбрать второй объект для сравнения
		найти связь объекта-признака со вторым объектом сравнения
		
	2 вариант : сравнение объектов по конкретному признаку (поиск общего признака у объектов)
		выбрать для сравнения объект-признак (класс)
		выбрать первый объект для сравнения
		выбрать второй объект для сравнения
		найти связь объекта-признака с первым объектом сравнения
		найти связь объекта-признака со вторым объектом сравнения
		
	3 вариант : сравнение конкретных объектов без точного признака (поиск общих признаков объектов)
		выбрать первый объект для сравнения
		выбрать второй объект для сравнения
		создать объект количества общих признаков (классов)
		найти объекты-признаки (классы), связанные с обоими объектами сравнения
			выбрать для сравнения объект-признак (класс) связанный с первым объектом
			найти связь объекта-признака со вторым объектом сравнения
			если связь существует, увеличить количество общих признаков (классов)
			выполнять цикл, пока есть объекты-признаки, связанные с первым объектом
		получить объект количества общих признаков
		чем больше количество общих признаков (классов), тем больше степень их схожести
